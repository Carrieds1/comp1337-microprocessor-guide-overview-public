# COMP1337 Microprocessor Guide

NOTE: Full materials are maintained privately and are available upon request for evaluation purposes.

A tutor-developed, 13-part learning resource covering machine code and microprocessor fundamentals, created to support beginner students in COMP1337.

---

## Overview

This repository contains a structured set of teaching materials designed to help students understand low-level programming concepts, including memory manipulation, function calls, and stack behaviour.

The content was developed while serving as a tutor for the course, with a focus on making complex topics more intuitive and accessible.

This guide was developed after completing the course, motivated by the difficulty many students (including myself) initially experienced in understanding microprocessor concepts.

The aim was to create the kind of resource I wished existed at the time, one that breaks down complex ideas into clear, practical mental models.

---

## Contents

### Microprocessor Tips (Core Resource)
- A 13-part structured series covering:
  - Machine code fundamentals
  - Instruction sets (4001, 4002, 8003, 8604, 8605)
  - Stack operations and memory reasoning
  - Function calls and frame pointers
  - Problem-solving strategies
- Designed as a combination of:
  - guided explanations  
  - alternative approaches to understanding instruction usage  
  - bite-sized clarifications for commonly misunderstood concepts  
- Includes:
  - a detailed walkthrough of the thought process used to solve microprocessor challenge problems  
  - translation of microprocessor instructions into C to support COMP1511 learning  

### Optional Lesson
- Post-assignment breakdown of common student mistakes
- Q&A-style explanations of key concepts:
  - QDECL (COMP1337 calling convention, equivalent to CDECL)  
  - Stack frames and memory layout  
  - Debugging approaches  
- Includes translation of assignments from the course’s microprocessor model to MIPS-style representations to support progression into COMP1521

This lesson was designed based on the belief that connecting concepts reinforces understanding, and that comprehensive, structured feedback is essential for developing strong problem-solving and reasoning skills

---

## Purpose

This guide was created to address gaps in understanding observed during both personal learning and tutoring.

The goal was to:
- simplify abstract concepts into clear mental models  
- provide structured progression through topics  
- bridge the gap between theory and practical implementation  
- support debugging and reasoning, not just memorisation
